Communication between two boards is done in following format:

- Main board starts from listening
- Remote board once every 3 sec sends status to Main board
    - when some data is intercepted during wait period -> restart timer
  - status consists of two bytes ending with CRC-8 byte
    - information to send consists of 4 flags, that are encoded into three bytes
      - f1 - power on request
      - f2 - fan on request
      - f3 - heat request
      - f4 - fire alarm!
    - B1=[f1,f2,f3,f4,!f1,!f2,!f3,!f4]
    - B2=[!f1,!f2,!f3,!f4,f1,f2,f3,f4]
- Main board as soon as receives data, waits for 10ms and replies with:
  - ACK 0x66 - if CRC and data matches template
  - CRC_ERR 0x96 - if CRC matches
  - DATA_ERR 0x99 - if CRC matches however data not
- If Remote board receives:
  - ACK - success (wait for 3 sec)
  - CRC_ERR,DATA_ERR,anything else - failed - retry sending after 10ms delay
  - timedout (200ms) - resend again (up to 3 retries, when retries end - wait 3 sec after last intercept)
